## 1. 产品定位

> **SeichiGo 是一个圣地巡礼旅行工具，用手帐的视觉与叙事让它有情感和留存。**

**主线**: 实用工具（地图 / 路线 / 攻略 / 导航 / 打卡）。
**情感线**: 手帐式视觉、文案、动画、累积感。

**冲突时永远工具优先。** 手帐不能让用户在 UI 上猜任何按钮。手帐活在视觉、文案语气、动画反馈、长期累积感里，**不在 UI 命名上**。

---

## 2. Manifesto · 100 字以内的定海神针

> **Seichigo 是一本你和最爱的作品共同书写的日本旅行手帐。**
>
> 每一部动画是一张地图，每一个取景地是一页空白，等你去那里盖下印章。我们整理全球十万处次元坐标，让你探索、规划、走过、回望；你贴上自己的照片，写下自己的日期，留下自己的路线。
>
> 一年后翻开它——那不是一份数据导出，是你活过、走过、爱过的全部证据。

**主 Slogan**: 你最爱的作品，写自己的手帐。
**副 Slogan**: 从屏幕，走向现实。
**English**: *An anime pilgrimage journal. Yours, page by page.*

## 3. 五条设计原则

| # | 原则 | 反例（违反就拒掉）|
|---|------|----------------|
| 1 | **工具命名清楚，情感视觉浓郁** | 按钮叫"朱印"、"誊抄"这种需要解释的词 |
| 2 | **每一个数据点都带故事文案** | 列表卡片只显示坐标和距离 |
| 3 | **轻动作通向重动作** | 把收藏 / 规划 / 打卡 / 分享做成 4 个平级 Tab |
| 4 | **仪式感在动画里，不在按钮上** | 打卡完只 toast 一个"成功" |
| 5 | **沉淀感在「我的手帐」页里集中爆发** | 工具页塞满"我的"信息，"我的手帐"反而空 |

## 4. 产品词汇表

### 必须使用 (UI 命名 / 文案默认词)

| 概念 | 用词 | 说明 |
|------|------|------|
| 用户主页 | **我的手帐** | 登录后默认首页 |
| 待办路线 / 旅行计划 | **行程** | 用户的真实物件 |
| 草稿行程（默认）| **我想去的地方** | 一切 ❤️ 收藏的默认归处 |
| 轻收藏动作 | **想去** / ❤️ | 永远落入草稿行程 |
| 现场打卡 | **打卡** | 不叫"朱印"、"签到" |
| 复制路线 | **加入我的行程** / **复制行程** | 不叫"誊抄" |
| 照片库 | **我的相册** | 不叫"对屏录" |
| 时间轴 | **我的旅程表** | 不叫"打卡时间轴" |
| 徽章 | **成就墙** | 不叫"印记墙" |
| 个人随笔 | **我的随笔** | 复用 Article 表，可设私密/公开 |
| 个人足迹地图 | **我走过的地方** | dashboard 板块名 |
| 社区 | **社区** | 内容 = 公开的其他用户的手帐 |
| LBS 入口 | **看看身边有什么** / **附近** | 永远是右下浮层，不进顶部 Nav |

### 永远不要使用

| 自造 / 文言词 | 已被否决，原因 |
|---|---|
| 朱印 / 盖印 | 用户不懂 |
| 誊抄 | 需要场景才能理解 |
| 想去笺 | 多余的字 |
| 对屏录 | 完全自造 |
| 手帐市 | 不直观 |
| 路书 | 过于官方 / 旅游业内黑话 |
| 番剧 (作 UI 入口) | 旧动漫圈语言；作品概念已并入地图筛选 |

### 黄色名单 (允许出现在内容文案，不允许出现在 UI 按钮)

| 词 | 出现位置示例 |
|---|---|
| 朱印 | 「这次旅行获得了 8 枚朱印」← 这种叙述句 OK |
| 巡礼者 | 头像旁的小印章「#0042 号巡礼者」← 装饰 OK |
| 巡礼 | 副标 / 内容 OK，按钮不用 |

## 5. 信息架构 (双界面)

### 5.1 双界面是产品形态的核心

```
未登录用户 → 首页 = Landing (工具索引 + 情感预告)
登录用户   → 首页 = 我的手帐 (个人空间中枢)
```

**这不是表面差异，是产品定位的核心。**
工具产品看 session；手帐产品看生命线。登录用户每次回来第一眼看见的是**自己的累积**。

### 5.2 顶部 Nav (6 项，全状态可见)

```
圣地巡礼 ⛩  |  首页 · 目的地 · 路线 · 攻略 · 社区 · 关于我们  |  🔍 🔔 [头像 ▾]
```

- **目的地** = 地图 (含按作品筛选 + 精简模式 + 街景)
- **路线** = 编辑部精选路线 + 高赞社区路线
- **攻略** = 配机位 / 时间戳 / 对比图的结构化向导
- **社区** = 别人的公开手帐索引
- **首页** = 未登录 Landing / 登录"我的手帐"，状态自动切换

**右下角常驻浮层**: ◎ 符号 + 红色脉冲点 = **看看身边有什么** (LBS)。不进顶部 Nav。

### 5.3 「我的手帐」内部 5 个章节

```
封面区 (累积数据 + 当前行程 + 头像 + VOL.01 第 N 天)
  ├─ 我走过的地方       (个人地图)
  ├─ 我的随笔           (写文章 / 日记)
  ├─ 我的旅程表         (横向时间轴)
  ├─ 作品巡礼进度       (按作品的完成度条)
  ├─ 成就墙             (徽章)
  ├─ 出行方式分布       (电车 / 巴士 / 自驾 / 徒步)
  └─ 我的相册           (照片 + 自动拼图)

底部: 探索区 (4 张卡 → 派出去到 4 个工具)
```

每个章节支持**独立的公开 / 私密开关**。

## 6. 5 种内容入口 × 4 种用户动作

整个产品在做的事，是把右边的"动作"沉淀进用户的手帐，途径是左边的"入口"。

### 6.1 5 种内容入口 (从工具向手帐注入数据)

| 入口 | 心理开场白 | 顶部 Nav 位置 |
|------|------------|---------------|
| **目的地** | "我想去日本/某城市" | 顶部 Nav |
| **路线** | "给我一条现成的行程" | 顶部 Nav |
| **攻略** | "我想深度学这部作品" | 顶部 Nav |
| **社区** | "看看别人怎么走的" | 顶部 Nav |
| **附近** | "我已经在这里了" | 右下浮层 |

### 6.2 4 种用户动作 (在手帐中沉淀)

| 动作 | 沉淀形态 | 现有代码 |
|------|----------|----------|
| ❤️ **想去** | 进入默认草稿行程「我想去的地方」 | `Favorite / MdxFavorite / AnitabiFavorite` → W2 合并 |
| 📒 **行程** | 一份可编辑的多日规划 | `RouteBook + RouteBookPoint` |
| 📷 **打卡** | 时间轴上的一个红点 + 照片 + GPS | `UserPointState` |
| 🎨 **自动拼图** | 相册里的一张对比图 | (W5 新增 AI 匹配) |

### 6.3 ❤️ 收藏的默认归宿 (重要决定)

任何 ❤️ 行为（不论收藏作品、地点、路线、攻略）**统一进入用户默认的草稿行程 "我想去的地方"**。
该草稿行程对每个新用户在注册时自动生成、永远存在、不能删除。
当用户准备真去时，把它拆成具体的「京都 5 日行程」等等。

→ 这是手帐"轻动作通向重动作"原则的落地：收藏不是一个孤儿数据，是行程的草稿。

## 7. 手帐"舍不得丢"的 5 个粘性支点

每个手帐页的实现必须每一条都被踩中：

1. **物理质感** — 纸张纹理、墨水晕开、印章按压、翻页动画
2. **个人化** — 封面 / 章节顺序 / 字体可换（变现合理边界在此）
3. **累积可见** — 数字必须显眼且持续增长（公里、打卡数、作品数）
4. **时间锚定** — 每一条记录都带日期，越老越有重量
5. **不可替代** — 别的平台无法导入这本手帐，只能在 Seichigo 续写

## 8. 视觉系统

### 8.1 色系

| 用途 | HEX | 备注 |
|------|------|------|
| 纸张底色 | `#f3ecdc` | 温暖象牙白，有轻微泛黄 |
| 浓墨 | `#1f1a13` | 主文本 |
| 墨柔 | `#4a4236` | 次文本 |
| 墨淡 | `#847b6c` | 弱化文本 / 说明 |
| 朱印红 | `#a8392b` | 印章主色 / 强调 / 当前态 |
| 朱印深 | `#862c20` | 渐变深端 |
| 墨蓝 | `#2d3e50` | 次强调 |
| 线缝灰 | `rgba(31,26,19,0.18)` | 虚线 / 分隔 |

### 8.2 字体

| 用途 | 字体 |
|------|------|
| 标题 / 数字 | **Noto Serif SC** (宋体，加粗) |
| 正文中文 | Noto Sans SC |
| 装饰英文 | *Cormorant Garamond Italic* |
| 手写按钮 | **Ma Shan Zheng** |

### 8.3 视觉元素 (强标识)

- **红色朱印章**：方印 + 圆印混用，微旋转 -4° ~ +3°，每页 4-6 处，**禁止滥用**
- **和纸胶带**：斜贴在卡片顶部，朱红 / 琥珀 / 苔绿 / 紫罗兰随机
- **缝纫虚线边框**：用于"贴上去"的卡片
- **墨笔分割线**：左右淡出，中段稳定
- **水彩底纹 + SVG 噪点**：纸张质感的底层
- **页角折角阴影**：每张 paper 卡片右下角
- **页面装订线**：左侧极淡的点状虚线，暗示这是一本本子
- **樱花枝 / 竹叶 / 山岚**：边角装饰，每页最多 1-2 处

## 9. 6 周 MVP 节奏

按这个顺序严格走，因为每一步是下一步的前提。

| 周 | 主题 | 核心交付 |
|---|------|----------|
| **W1** | 「我的手帐」首页雏形 | 登录后首屏 = 手帐。封面 + 5 个章节卡。**做出物理纸质手帐的视觉质感**。|
| **W2** | 数据双向打通 + 打卡仪式 | 合并 `Favorite/MdxFavorite/AnitabiFavorite` 为统一「想去」，默认进入草稿行程「我想去的地方」。打卡完成弹出"打卡卡片"（红色印章动画 + 作品 + 地点 + sync rate），自动入册。|
| **W3** | 行程 = 新一代路书 | 重命名 `RouteBook` 产品语为"行程"。在地图 / 路线 / 攻略页加"加入我的行程"按钮。路书底部永久显示"复制自 XXX"。|
| **W4** | 首页双形态切换 | 未登录 = Landing；登录 = 我的手帐。这一周把整套体系对所有用户上线。|
| **W5** | 自动拼图 + 相册 | AI 截图匹配，生成对比图，自动进相册章节。|
| **W6** | 公开手帐 + 社区 v1 | 每章节加公开开关。社区 = 别人公开手帐的索引流，可"复制行程"。|

**W7+ (未排):** 年报、付费定制封面 / 字体、纸质打印。

## 10. 反 Feature List (明确不做)

| 不做的事 | 原因 |
|---|---|
| 「作品 / 番剧」作为顶部 Nav 独立 Tab | 已并入地图筛选 + 精简模式 |
| 「手帐」作为顶部 Nav 独立 Tab | 登录后即首页，不需要二次入口 |
| 「朱印 / 誊抄 / 想去笺 / 对屏录 / 手帐市」等自造词出现在 UI | 强制每个按钮看一眼就懂 |
| 通用 UGC 社区池 (类似小红书流) | 社区只做"翻别人的手帐" |
| 任何"解锁更多功能"型 paywall | 付费只能买"让我的手帐更像我" |
| 任何打断浏览的广告 | 工具型产品的底线 |
| 用户手帐导出 (JSON / PDF / 等) | 这是温柔锁定，不导出是产品策略 |
| 把"探索 / 规划 / 导航 / 分享"做成 4 个平级 Tab | 它们是阶段不是位置；不能并列 |
| 把"想去"做成沉淀型数据（独立 wishlist 表）| ❤️ 永远进入默认草稿行程 |
| 中英日翻译质量妥协 (auto-only) | 已有 Gemini 翻译队列 + 人工校审 (lib/translation/) |

## 11. 变现合理边界

**变现方向 = "让手帐更像你"**，不是"解锁更多功能"。

可以做：
- 封面 / 装订 / 字体 / 章节顺序的付费定制模板
- 把手帐打印成实体本（年终版 / 纪念版）
- 自动生成"巡礼年报"（免费基础版 + 付费深度版）
- 与日本本地 JR / 旅馆 / 文创周边的非侵入式合作链路

不做：
- 任何功能性 paywall (探索 / 规划 / 导航 / 拼图都免费)
- 任何打断浏览的广告

## 附录 A: 决策来源

本宪章基于 2026-05-13 的产品对话整理。关键决策点：

1. ❌ 产品 ≠ 地图工具 + CMS。✅ 产品 = 旅行工具 + 手帐叙事。
2. ❌ 用图鉴 / 库 / 池 作为统一容器。✅ 用"作品 / 目的地 / 路线 / 攻略"四个入口 + "我的手帐"沉淀地。
3. ❌ 一界面（未登录 / 登录都看 Landing）。✅ 双界面（未登录 Landing / 登录手帐）。
4. ❌ 用诗意自造词（朱印 / 誊抄）。✅ 用直觉词（打卡 / 加入行程）。
5. ❌ "想去"做成存储型 wishlist。✅ "想去"是动作 + ❤️ 落到默认草稿行程。
6. ❌ "作品"作独立 Tab。✅ 并入"目的地"地图筛选。
7. ❌ "手帐内页 / 打卡时间轴 / 印记墙 / 最近对比图"等开发者视角命名。✅ "我走过的地方 / 我的旅程表 / 成就墙 / 我的相册"。
8. ❌ 5 数据格用"县 / 巡礼地 / 打卡点 / 照片 / 手帐记录"。✅ 5 数据格用"部作品 / 取景地 / 次打卡 / 公里 / 次行程"——"公里"是手帐情感的关键沉淀。
